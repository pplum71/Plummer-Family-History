Plummer Family History — Quick Start

  1. Edit data in /data/*.json.
  2. Replace the crest at /assets/logo.svg with your own.
  3. Formspree: create a form and swap in your ID in contact.html.

Deploy to GitHub Pages

1. Create a repo named e.g. plummer-family-history
2. Upload all files from this zip (keep the folders)
3. In repo settings → Pages → Deploy from branch → main / root
4. Wait 1–2 minutes, then open your site URL

Deploy to Netlify

1. Drag & drop this folder into Netlify
2. Or connect your Git repo; build command: (leave blank), publish dir: root

Hide People from public nav

Open any page and remove the People link in the nav. Editors can still access /people.html directly or via Admin page.

Privacy Notes

  • Exclude birthdates/locations for living relatives unless you have permission.
  • Use first names or initials if needed.

Admin Editors

  • People Manager: add/edit relatives, export people.json.
  • Stories Manager: write stories/recipes, export stories.json.
  • Photos Manager: enter photo URLs + captions, export photos.json.

Open /admin.html?key=YOURSECRET (change YOURSECRET in the file). Drafts auto‑save in your browser.

Direct GitHub Commits from Admin

  1. Create a fine-grained Personal Access Token limited to this repo with contents:write; set an expiration.
  2. Open /admin.html?key=YOURSECRET → fill in Owner, Repo, Branch, Images Path, and Token → Save.
  3. Photos Manager: Choose Images → Upload to GitHub & Add (adds to assets/photos and inserts raw URLs).
  4. Click Commit photos.json to GitHub (and similarly for people/stories) to publish JSON updates.

Your token is stored only in your browser (localStorage). For production, consider using Netlify Functions with server-side tokens for better security.

D3 Tree (SVG)

The Family Tree page includes a collapsible D3 SVG tree built from people.json (using parents arrays). You can customize the layout in family-tree.html near the "D3 Tree (SVG)" section.
